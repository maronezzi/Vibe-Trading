🔄 vt_config_loader.py — Hot Reload Gateway

109 linhas — detecta mudanças no JSON e recarrega sem reiniciar o bot

Como funciona o Hot Reload

📁
1. Checa mtime
os.path.getmtime() — timestamp da última modificação do arquivo
🔍
2. Comparar
mtime == cache? → Cache hit (zero custo). Mudou? → Recarrega
✅
3. Validar
Checa se keys obrigatórias existem: symbols, strategy, wdo, win
♻️
4. Atualizar
Substitui cache global + log: "🔄 v48→v49 (by optimizer)"

📊 BOLLINGER

      
Bandas de Bollinger (10, 1.5) + RSI extremo. Compra no lower band + RSI oversold. Vende no upper band + RSI overbought.

      
Usado em: 6 TFs (WIN×3, IND×2, WIN_H1)

    

    

      
📈 RSI_REVERSION NOVA

      
Mean-reversion pura por RSI. Compra quando RSI < oversold, vende quando RSI > overbought. Sem Bollinger, sem volume — entrada mais cedo.

      
Usado em: 9 TFs (BIT, DOL, IND, WSP, WDO, WIN_M30)

    

    

      
📉 EMA_PULLBACK

      
Pullback na EMA em tendência. EMA fast/slow + ADX > threshold. Entra quando preço volta pra EMA após rompimento.

      
Usado em: 7 TFs (BIT_M15, DOL×2, IND_H1, WSP_M30, WDO×2)

    

    

      
📊 VWAP

      
Volume Weighted Average Price. Compra acima do VWAP (momentum), vende abaixo (reversão). Parâmetros: period, buy/sell threshold.

      
Usado em: 2 TFs (BIT_H1, DOL_M30, WSP_H1)

    

    

      
💪 STRONG_TREND

      
Seguidor de tendência forte. Detecta movimentos com momentum alto e entra na direção. Sem parâmetros extras — usa só ATR e preço.

      
Usado em: 1 TF (WSP_M15)

🐛 Bug Crítico Corrigido Hoje

  
Case mismatch: os 24 timeframes operariam com SL errado sem esta correção


  

    

      
❌ ANTES (bug)

      

        
_get_params_for_tf("WIN", "M5")

        
  key = "win_m5" ← minúsculo!

        
  params_by_tf["win_m5"] → NÃO EXISTE

        
  chave real é "WIN_M5" (maiúsculo)

        
  → fallback: SL=0.8x (genérico)

        
TODOS os 24 TFs ignoram params otimizados!

      

    

    

      
✅ DEPOIS (fix)

      

        
_get_params_for_tf("WIN", "M5")

        
  key = "WIN_M5" ← original

        
  params_by_tf["WIN_M5"] → ENCONTRADO

        
  fallback case-insensitive também

        
  → merge: SL=2.5x + trailing + cooldown

        
Todos os 24 TFs recebem params corretos!
